# Phoenix Journeys

**世界很大，从一段故事开始。**

Phoenix Journeys 是一款故事驱动的世界探索产品。

探索者先被人物、地点、历史细节与未解问题吸引，再在旅途中自然接触语言、文化和知识。语言学习是探索产生的结果，不是要求用户每天完成的任务。

## 产品定位

- 不是课程软件，也不是签到、刷题或每日任务工具。
- 不是只提供景点资料的普通旅游导览。
- 每段 Journey 必须先有值得追下去的故事，再提供按需出现的语言帮助。
- AI 的角色是同行者和提问者，不是一直批改用户的老师。
- 地图、飞机、声音、背景、印章与互动都服务于沉浸和好奇心。

## 当前阶段

- 平台：Flutter 移动优先 PWA，最后包装为 iOS 与 Android App
- 发布：Cloudflare Workers
- 内容：7 座城市、8 个地点，支持同一城市继续扩展多个地点
- 体验：故事、朗读、点词解释、Discovery、思考、表达、回忆、分享和城市印章
- AI：PhoenixGuideAgent、PhoenixWritingAgent 与统一 AI 调度架构
- 视觉：每个地点独立离线背景库，重点 Journey 使用低干扰电影感动态背景
- 状态：真实产品持续开发与 Founder 手机验收，不制作独立 Demo

## 核心原则

- **故事优先**：先制造好奇，再展开知识与语言。
- **学习隐形**：拼音、母语、英文、词义和写作帮助只在需要时出现。
- **事实可信**：历史、文化、人物和科学内容必须有审核来源；传说与争议必须明确标注。
- **视觉合法**：图片、字体、音乐、插图和 UI 资源必须原创或拥有合法授权。
- **沉浸克制**：动态效果不能抢夺故事阅读，不使用廉价、重复或高成本动画。
- **无强迫留存**：不加入签到焦虑、金币诱导、连续天数惩罚或第三方广告。
- **隐私优先**：个人回忆和探索记录默认保存在客户端，服务器不默认持久保存。
- **稳定发布**：新功能先进入独立分支和 Preview，手机确认后才合并到 `main`。

## 当前产品重点

1. 把固定课程感步骤升级为故事线索与探索动作。
2. 为每段 Journey 建立悬念、人物、冲突、发现和余韵。
3. 加入轻量分支，让探索者的选择改变看到的故事与知识。
4. 把 AI 导游升级为会记住兴趣的同行者。
5. 让同一城市的不同地点组成可连续追踪的故事宇宙。
6. 完成剩余地点动态背景，同时坚持慢镜头、分层光影、减少动态回退和手机性能保护。

## Repository Structure

- `app/` Flutter PWA 与未来 App 客户端
- `backend/` 数据库与服务设计
- `ai/` AI Prompt、调度、审核与交互规范
- `content/` Journey 故事、词汇、来源和本地化资料
- `design/` 设计系统、原创素材规范和授权记录
- `docs/` 产品方向、开发流程、验收规范和决策记录
- `roadmap/` 阶段路线图
- `.github/` PR Preview、CI 与发布流程
